# 🜲 MOTHERLODE — LobangKing.sg blueprint

> Every Deal in Singapore — verified daily.  
> **Static, automated, AI-assisted deals website** · blueprint v1.0

This is the master knowledge base for this site — written so a human **or another AI**
can rebuild the whole system on a new site. The machine-readable twin is
[`MOTHERLODE.yaml`](MOTHERLODE.yaml); an LLM index is [`llms.txt`](llms.txt).

## Stack
- **frontend**: Hand-written HTML + CSS + vanilla JS (no framework, no build for the browser)
- **hosting**: GitHub Pages (free static hosting) + optional Cloudflare (WAF, headers, analytics)
- **automation**: GitHub Actions (free scheduled jobs) — the 'backend' for a static site
- **data**: A single JSON file (data/deals.json) acts as the database
- **build**: Dependency-free Python (standard library only) — runs on the GitHub runner
- **ai**: Optional server-side LLM enrichment + optional on-device browser AI (translation)

## Architecture
- No server, no database, no framework — just files on a CDN. This is the core security + scaling advantage.
- GitHub Actions act as the backend: scheduled scripts fetch data, build pages, validate, and commit; GitHub Pages redeploys on each commit.
- Content is PRE-RENDERED into static HTML for SEO, then the same content is hydrated by JS for interactivity.
- Single sources of truth (category list, expiry parser, page builder) are imported across scripts so behaviour can't drift.

## Components
### `index.html, deals.html, about.html, submit.html, privacy.html, 404.html`
_html_ — The site pages. Share one header/footer (generated by build_pages) and a strict CSP.
  ↳ Deal grids are pre-rendered between <!--DEALS-->/<!--SPOT--> markers then hydrated by main.js.

### `css/styles.css (+ styles.min.css)`
_css_ — Full design system: CSS custom-property tokens, light+dark themes, components, utilities.
  ↳ Minified to .min.css by minify.py during the build. No inline styles anywhere (CSP style-src 'self').

### `css/fonts.css, fonts/`
_css_ — Self-hosted @font-face (Sora + DM Sans). No external font requests.
  ↳ Run scripts/get_fonts.sh once to download the woff2 files; falls back to system fonts until then.

### `js/theme.js`
_js_ — Sets light/dark on <html> before paint to avoid a flash. External file so the CSP can stay strict.

### `js/main.js`
_js_ — Render deals from JSON, live search/filter/sort, paginated 'load more', vote, share, copy code, theme toggle, mobile menu, back-to-top, service-worker registration.
  ↳ All HTML writes go through one Trusted-Types-safe sink (setHTML). Re-applies translation after re-render.

### `js/vitals.js`
_js_ — Core Web Vitals RUM (LCP/CLS/INP/TTFB) via PerformanceObserver. No external calls.

### `js/engagement.js`
_js_ — PWA install prompt + 'deals refreshed since your last visit' badge (return-visit drivers).

### `js/protect.js`
_js_ — Deterrents: invisible content canary, console copyright notice, image drag/right-click block (images only).

### `js/translate.js`
_js_ — On-device AI translation (Chrome built-in Translator / Gemini Nano) into 中文/Malay/Tamil. Private, free, no API key. Progressive — hidden if unsupported.

### `sw.js`
_js_ — Offline + instant repeat visits. Static assets cache-first; deals.json network-first. Bump CACHE name on asset change.

### `scripts/aggregate_deals.py`
_py_ — DAILY: fetch deals from RSS feeds + per-brand Google News + optional affiliate API (concurrently), categorise across all categories, detect brands, parse expiry, dedupe, drop expired-at-fetch, optional AI polish, write deals.json.
  ↳ Never invents data; every deal keeps a source link. Single source of truth for CATEGORIES, BRANDS and parse_expiry.

### `scripts/ai_enrich.py`
_py_ — OPTIONAL: polish titles/summaries/categories with any OpenAI-compatible LLM, using ONLY the fetched text (never invents). Safe no-op when unconfigured.

### `scripts/build_pages.py`
_py_ — Single-source header/footer; pre-render deals + JSON-LD into HTML; generate per-deal (deal-<id>.html) and per-category (cat-<id>.html) SEO pages with Offer/ItemList schema; minify CSS; write feed.xml + sitemap.xml; render only ACTIVE (non-expired) deals; prune orphan pages.

### `scripts/minify.py`
_py_ — Conservative CSS minifier (comments + whitespace only — never removes meaningful spaces).

### `scripts/validate.py`
_py_ — HOURLY watchdog: validates JSON/XML, security/CSP compliance, asset integrity; runs the retention GC; rebuilds. Exits non-zero on compliance errors (alerts owner).

### `scripts/prune_pages.py`
_py_ — Hard-deletes deals expired >10 days from the database, then rebuilds so the sitemap + per-deal pages auto-sync (new deals get URLs; expired/orphan pages deleted).

### `scripts/check_links.py`
_py_ — DAILY: removes deals whose outbound URL is definitively dead (404/410/451 only; ignores transient errors), then rebuilds.

### `scripts/snapshot.py, history/`
_py_ — DAILY: dated snapshot of deals.json + an added/removed change-log; prunes to the last 60 days (audit trail + rollback).

### `scripts/test_aggregate.py`
_py_ — Offline unit tests: parsing, expiry-drop, dedupe, categorisation, brand detection, AI no-op.

### `scripts/sources.json, scripts/manual_deals.json`
_config_ — sources.json: RSS feeds, brand watchlist, AI + affiliate toggles, settings. manual_deals.json: always-on curated deals.

### `robots.txt, sitemap.xml, feed.xml, manifest.webmanifest`
_config_ — robots blocks AI/scraper bots (keeps search engines); sitemap+feed auto-generated; manifest = installable PWA.

### `_headers, .well-known/security.txt`
_config_ — _headers: full security header set for Cloudflare/Netlify (+ CSP report-only Trusted Types). security.txt: responsible-disclosure (RFC 9116).

### `.github/workflows/{lighthouse,a11y,codeql}.yml, .github/dependabot.yml, lighthouserc.json, .pa11yci.json`
_ci_ — Quality/security gates: Lighthouse budget, pa11y WCAG-AA, CodeQL JS scan, auto-patched Actions.

### `.github/workflows/{update-deals,validate,health-check,check-links}.yml`
_ci_ — Scheduled jobs: daily deals build, hourly validate+GC, 30-min uptime ping, daily dead-link prune.

## Security posture
- **csp**: default-src 'self'; script-src 'self'; style-src 'self' (NO unsafe-inline anywhere); connect/font/img/manifest/worker 'self'; form-action 'self' + formspree.io; object-src 'none'; base-uri 'self'; frame-ancestors 'none'; upgrade-insecure-requests; report-to csp.
- **trusted_types**: All DOM HTML writes go through one audited sink (setHTML) + a TrustedScriptURL policy for the SW. Shipped report-only; flip require-trusted-types-for 'script' into the enforced CSP after a browser test.
- **headers**: HSTS preload, X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy, Permissions-Policy, COOP, CORP, CSP reporting endpoints.
- **xss**: Every value escaped (esc) + link scheme-checked (safeUrl) before render; feed-sourced data treated as untrusted.
- **secrets**: None in the browser. Affiliate/AI keys live only in GitHub Secrets, used server-side in Actions.
- **anti_abuse**: Form honeypot + time-trap; AI/scraper bots blocked in robots; Cloudflare WAF/bot-fight/rate-limit recommended.
- **anti_copy**: Honest stance: public content is copyable. Real moat = daily-rebuilt content (clones go stale in 24h) + canary + Cloudflare + DMCA.

## Data model
- **file**: data/deals.json
- **shape**: { updated, categories[], brands_tracked[], brands_today[], deals[] }
- **deal_fields**: id, title, store, categories[], icon, badges[{type,label}], access{type,label}, code?, desc, expiry, heat, url, source{name,url}, brand?, spotlight?
- **categories**: all, food, electronics, home, travel, transport, fashion, entertainment, finance, online
- **expiry_rule**: parse_expiry distinguishes start dates ('from'), end keywords ('till/until/ends'), ranges, single days; a deal is ACTIVE while end>=today.

## Reusable patterns
- Static site + CDN + GitHub Actions = a free, daily-fresh site that feels dynamic with zero server.
- Single source of truth: import CATEGORIES, parse_expiry and the builder everywhere so logic never drifts.
- Generate on the server (Actions), keep the browser lean; pre-render for SEO + hydrate for interactivity.
- Lock the CSP to 'self', make code Trusted-Types-ready, ship risky enforcement report-only first.
- Progressive enhancement: every advanced feature (translate, view transitions, scroll anim) degrades gracefully.
- Never fabricate data; if using AI, only summarise/translate REAL fetched text and keep the source link.
- Idempotent builders + diff-guarded commits + deterministic output = no spurious churn.
- Retention model: hide expired content immediately for UX, hard-delete after a grace window for the DB.
- An hourly validator that auto-fixes data and fails loudly on compliance regressions keeps the site honest.

## Key decisions
- **Self-host fonts** — Privacy + performance + lets the CSP drop external origins.
- **Remove all 'unsafe-inline'** — A+ CSP; inline styles moved to utility classes.
- **Trusted Types report-only (not enforced)** — Can't browser-test here; enforcing blindly could break JS for Chrome users. One-line flip after a DevTools check.
- **JSON file as database** — No server needed; diffable, versioned, auditable in git.
- **Provider-agnostic AI (env-configured)** — No vendor lock-in; free providers (GitHub Models/Groq/Gemini) swap via env.
- **Formspree for forms** — Static sites can't process POST; one scoped form-action CSP exception.
- **Per-deal + category pages** — Long-tail SEO — each deal becomes its own indexable URL.

## Rebuild this for a NEW site
1. Copy the repo structure (scripts/, js/, css/, .github/, pages, config files).
2. Rebrand: replace the logo and run the icon generator; change --navy/--gold tokens in css/styles.css; set SITE_URL + name in scripts/build_pages.py and aggregate_deals.py.
3. Re-point the data source: edit scripts/sources.json (RSS feeds, brand watchlist) — or replace fetch_rss with your own domain's data source.
4. Adjust CATEGORIES and BRANDS in scripts/aggregate_deals.py to the new niche.
5. Rewrite the page copy (hero, about, privacy) and the JSON-LD names.
6. Set integrations: Formspree form ID, social handles (SOCIALS in build_pages), security.txt contact, and optional AI/affiliate GitHub Secrets.
7. Deploy: push to GitHub, enable Pages (deploy from branch / root), enable Actions, optionally put Cloudflare in front.
8. Verify: run scripts/test_aggregate.py, scripts/build_pages.py and scripts/validate.py; then let the daily/hourly workflows take over.
